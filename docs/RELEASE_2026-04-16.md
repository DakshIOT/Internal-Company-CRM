# Release Notes - 2026-04-16

This note captures the changes completed locally on `2026-04-16` before live deployment.

## Functional Changes

- Fixed service edit/save amount handling when the existing rate is already formatted.
  - Edit forms now use plain decimal input values like `1000.00` instead of formatted values like `1,000.00`.
  - Admin can now change notes, attachments, or other fields without having to retype the rate.

- Fixed Function Entry package/service resync behavior.
  - If an admin changes a service rate in master data, existing editable Function Entries now pick up the updated service rate when reopened.
  - Package totals and Function Entry totals now recalculate correctly after that resync.

- Fixed stale relation reuse during Function Entry total recalculation.
  - Recalculation now reloads current package/service-line state before summing totals.
  - This resolved incorrect `package_total_minor` values in employee-select and flat-rate service mode flows.
